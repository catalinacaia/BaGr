Abstract

Sa se conceapa o aplicatie Web capabila sa genereze ecusoane pentru diverse evenimente (conferinte, ateliere de lucru si altele) si/sau contexte (acces in cadrul unui obiectiv turistic, semnalarea apartenentei la o organizatie, companie, grup de interes,...), eventual cu precizarea rolului: vizitator, stagiar, personal administrativ, participant, invitat special, student, profesor etc. Se va oferi posibilitatea alegerii/editarii unei machete (predefinite sau specificate de utilizator) de generare a unui set de ecusoane, inclusiv cu precizarea temei vizuale -- e.g., cromatica, orientare, dimensiuni, fonturi, sigle, URL-uri obligatoriu incluse. Generarea suitei de ecusoane se va realiza si avand ca date de intrare numele, afilierea si statutul participantilor preluate dintr-un fisier transferat prin 'upload' in cadrul instrumentului (minimal, vor fi acceptate formatele CSV si XML). Ecusoanele vor fi disponibile ca documente HTML si sub forma grafica (e.g., PNG). Bonus: ecusoane generate in format SVG ori incluzand reprezentari QR Code.

Generalitati despre limbajul HTML

HyperText Markup Language (HTML) este un limbaj de marcare utilizat pentru crearea paginilor web ce pot fi afisate intr-un browser (sau navigator). Scopul HTML este mai degraba prezentarea informatiilor si paragrafe, fonturi, tabele s.a.m.d.; decat descrierea semanticii documentului.

HTML este prescurtarea de la Hyper Text Mark-up Language si este codul care sta la baza paginilor web.

Paginile HTML sunt formate din etichete sau tag-uri si au extensia ".html" sau ".htm". In marea lor majoritate aceste etichete sunt pereche, una de deschidere <eticheta> si alta de inchidere </eticheta>, mai exista si cazuri in care nu se inchid, atunci se foloseste <eticheta />. Navigatorul web interpreteaza aceste etichete afisand rezultatul pe ecran. HTML-ul este un limbaj care nu face deosebire intre litere majuscule si minuscule.

Pagina principala a unui domeniu este fisierul "index.html" respectiv "index.htm". Aceasta pagina este setata a fi afisata automat la vizitarea unui domeniu.

De exemplu la vizitarea domeniului www.nume.ro este afisata pagina www.nume.ro/index.html.

Unele etichete permit utilizarea de atribute care pot avea anumite valori:

			 <eticheta atribut="valoare"> ... </eticheta>

Componenta unui document HTML este:

  1. versiunea HTML a documentului
  2. zona head cu etichetele <head> </head>
  3. zona body cu etichetele <body> </body> sau <frameset> </frameset>

Toate paginile HTML incep si se termina cu etichetele <html> si </html>.

In interiorul acestor etichete gasim perechile <head>, </head> si <body>, </body>.

head contine titlul paginii intre etichetele <title> si </title>, descrieri de tip <meta>, stiluri pentru formatarea textului, script-uri si legsturi catre fisiere externe (de exemplu script-uri, fisiere de tip CSS sau favicon).

Etichetele de tip meta contin cuvinte cheie, descrierea paginii, date despre autor, informatii utile motoarelor de cautare si au urmatorul format:

			  <META NAME="nume" CONTENT="continut">

Exemplu: link catre un fisier extern CSS:

			  <link rel="stylesheet" type="text/css" href="css.css">

body gazduieste practic toate etichetele afisate de browser pe ecran.

Exemplu: o pagina HTML cu titlul Exemplu iar continutul Continut pagina

			  <html>
			  <head>
			  <title>Exemplu</title>
			  </head>
			  <body>
			  Continut pagina
			  </body>
			  </html>

Si in HTML poate fi introdus un comentariu, care bineinteles nu va fi afisat de browser:

			  <!-- comentariu -->

Generalitati despre limbajul PHP

PHP este un limbaj de programare. Numele PHP provine din limba engleza si este un acronim recursiv : Php: Hypertext Preprocessor. Folosit initial pentru a produce pagini web dinamice, este folosit pe scara larga in dezvoltarea paginilor si aplicatiilor web. Se foloseste in principal inglobat in codul HTML, dar incepand de la versiunea 4.3.0 se poate folosi si in mod "linie de comanda" (CLI), permitand crearea de aplicatii independente. Este unul din cele mai importante limbaje de programare web open-source si server-side, existand versiuni disponibile pentru majoritatea web serverelor si pentru toate sistemele de operare. Conform statisticilor este instalat pe 20 de milioane de site-uri web si pe 1 milion de servere web. Este disponibil sub Licenta PHP si Free Software Foundation il considera a fi un software liber.

Generalitati despre limbajul JavaScript

JavaScript (JS) este un limbaj de programare orientat obiect bazat pe conceptul prototipurilor. Este folosit mai ales pentru introducerea unor functionalitati in paginile web, codul Javascript din aceste pagini fiind rulat de catre browser. Limbajul este binecunoscut pentru folosirea sa in construirea siturilor web, dar este folosit si pentru accesul la obiecte incastrate (embedded objects) in alte aplicatii. A fost dezvoltat initial de catre Brendan Eich de la Netscape Communications Corporation sub numele de Mocha, apoi LiveScript, si denumit in final JavaScript.

CSV

Este un acronim pentru Comma Separated Values. Este un format de fisier nestandard, in care valorile continute sunt despartite prin virgula, dar nu numai. Separatorul folosit poate fi oricare alt caracter, dar de obicei acesta este virgula sau caracterul tab.

Acest tip de fisier stocheaza date tabulare, de tip caracter sau numere, sub forma de text (plaintext).

AJAX (Asynchronous JavaScript and XML)

Este o suita de tehnologii specifica programarii web la nivel de client (client side programming), folosita pentru transferul asincron de date intre aplicatii si server. Prin AJAX, transferul se face in mod asincron, fara ca aplicatia sa intre in proces de asteptare pentru finalizarea transferului de date. Odata initiata cererea catre server, aplicatia continua sa ruleze, iar transferul de date va avea loc in background, in momentul cand acest lucru va fi posibil. Dupa primirea datelor, aplicatia va fi instiintata ca transferul s-a realizat, iar datele sunt disponibile pentru a fi folosite. Transferul se face prin obiecte de tip XMLHttpRequest. XML nu este obligatoriu sa fie folosit, in practica formatul JSON este cel preferat.

Tehnologiile AJAX pot fi folosite in combinatie cu HTML si CSS, iar JavaScript si XMLHttpRequest object ofera o metoda de a schimba date in mod asincron intre browser si server evitand reincarcarea paginilor la finalizarea transferului. DOM (Document Object Model), accesat cu JavaScript, pentru afisare dinamica a datelor, permite utilizatorului sa interactioneze cu acestea.

AJAX nu este un nou limbaj de programare [1], ci o noua modalitate de a folosi standardele existente. Pe scurt, este arta de a schimba date cu un server, si a incarca date intr-o pagina web, fara a fi nevoie ca aceasta sa fie reincarcata.

Arhitectura

Schema modulelor functionale

Site-ul va avea pagini pentru fiecare functionalitate. Bara de meniu si Logo vor fi prezente pentru fiecare accesare a site-ului.

Pagina Home va contine o scurta descriere despre functionalitatea site-ului precum si un buton spre pagina de creare a ecusoanelor.

Pagina Despre va contine o descriere mai ampla despe functionalitatea site-ului.

Pagina Creare va contine formularul de completare a datelor ce vor fi puse in ecuson precum si optiunea de a incarca un logo in cadrul ecusonului si de asemenea si o optiune de salvare a ecusonului creat. Pagina va contine si un formular in care se va putea incarca un fisier cu date ce vor fi prelucrate in ecusoane. De asemenea pagina va mai contine si un scurt istoric al ecusoanelor.

Pagina Contact ofera posibiltatea de a trimite un mesaj catre administratorul site-ului.

Concluzii

In urma realizarii acestui proiect, se impun unele concluzii.
